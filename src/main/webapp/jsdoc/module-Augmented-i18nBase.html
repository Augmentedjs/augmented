Fork of Jquery i18n plugin turned component Load and parse message bundle files (.properties), making bundles keys available as javascript variables. i18n files are named .js, or _.js or __.js Where: The argument is a valid ISO Language Code. These codes are the lower-case, two-letter codes as defined by ISO-639. You can find a full list of these codes at a number of sites, such as: http://www.loc.gov/standards/iso639-2/englangn.html The argument is a valid ISO Country Code. These codes are the upper-case, two-letter codes as defined by ISO-3166. You can find a full list of these codes at a number of sites, such as: http://www.iso.ch/iso/en/prods-services/iso3166ma/02iso-3166-code-lists/list-en1.html Sample usage for a bundles/Messages.properties bundle: i18n.properties({ name: 'Messages', language: 'en_US', path: 'bundles' });
